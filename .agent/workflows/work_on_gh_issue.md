---
description: How to work on a GitHub issue
---

1.  **View the Issue**: Run `gh issue view <issue-number>` to understand the requirements.
    - Example: `gh issue view 110`
2.  **Update Task List**: Create or update `task.md` with the checklist items from the issue.
3.  **Create Plan**: Create `implementation_plan.md` detailing the changes.
4.  **Implement**: Follow the plan to make changes.
5.  **Verify**: Run tests and manual verification.
6.  **Close Issue**: (Optional) Close the issue if completed.
